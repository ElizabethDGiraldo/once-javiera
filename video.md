# Taller componentes del internet o la red.
## 1. ¿Cómo funciona internet?
### El propósito de los protocolos **TCP/IP**.

- **IP:** otorga una dirección numérica decimal con puntos única a cada dispositivo conectado a la red.

- **TCP:** sirve para repartir la información en pequeños paquetes y dividirlos según la categoría a la que pertenecen (archivo, Gmail, fotos, etc). 

### La función del sistema **DNS**

- Es el traductor para que cualquier persona pueda llegar al IP sin tener que memorizar puntos y números. Solo agregar caracteres fáciles de identificar y de poder encontrar el destino en la red. Por ejemplo: copiar Google, abrir el sitio web y así entrar a su dirección **IP** sin memorizarla.

### El rol del protocolo DHCP
- El **DHCP** le asigna automáticamente una dirección IP disponible a tu dispositivo para que no tengas que configurarla manualmente, y así no tener que estar configurando cosas técnicas, sino que se abra automáticamente el sitio que se visite o acción que se quiera ejecutar.

## 2. Protocolo HTTP y la Web 

### Los componentes básicos de una URL
- **Esquema (Protocolo):** https (es el método de envío seguro).

- **Dominio:** www.ejemplo.com (el nombre del sitio).

- **Ruta:** /productos (la "carpeta" o sección específica dentro del sitio).

###	La diferencia fundamental entre los verbos HTTP
- **GET:** Es para pedir información. Cuando entras a un blog para leer, estás haciendo un GET. Los datos suelen ir visibles en la URL.

- **POST:** Es para enviar información. Cuando escribes tu usuario y contraseña o subes una foto, usas POST. Es más privado porque los datos van "escondidos" en el cuerpo del mensaje.

###	El significado y propósito de los códigos de estado más comunes vistos en clase (200, 301, 404 y 500)
- **200 OK:** ¡Todo salió bien! Aquí tienes tu página.

- **301 Moved Permanently:** La página se mudó a otra dirección (redirección).

- **404 Not Found:** El clásico. El servidor funciona, pero lo que buscas no existe. 

- **500 Internal Server Error:** El servidor "explotó" o tuvo un fallo interno; no es tu culpa, es de ellos.
